## [2.2.0] - 2026-06-03

### Added

- **.gitignore-aware walking** ‚Äî directories matching `.gitignore` patterns are skipped during walk, never added to inotify
- **Path-level exclusions** ‚Äî `WithExcludePaths` excludes absolute paths (and subtrees) during walk
- **Inotify budget awareness** ‚Äî auto-detected from `/proc/sys/fs/inotify/max_user_watches`, override with `WithMaxWatches`
- **Content hashing** ‚Äî `WithContentHashing` populates `Event.Hash` with SHA-256 (capped at 10 MiB)
- **Self-healing** ‚Äî `WithSelfHeal(interval)` auto-retries failed watch registrations
- **FilterGitignore** ‚Äî filter events using `.gitignore` patterns from a repo root
- **Metadata filters** ‚Äî `FilterWithMeta` variants return match reasons for debugging
- **Exponential backoff middleware** ‚Äî growing drop windows after failures
- **Prometheus collector** ‚Äî zero-dependency metrics collector
- **OpenTelemetry middleware** ‚Äî zero-dependency tracing via `OTelSpan` interface
- **Reset()** ‚Äî clears runtime state while preserving configuration
- **ENOSPC handling** ‚Äî graceful degradation instead of crashing
- **Batched registration** ‚Äî directories added in batches of 1000 for large trees
- **7 godoc examples** ‚Äî runnable examples in `examples/`

### Changed

- Rate limit delegate now uses throttle internally
- Unified `addPath` function for all path additions
- Generic `makeSetFilter` for extension/dir filters

### Fixed

- `.gitignore` ancestor matching
- Watch budget check on all `addPath` calls
- Duplicate root paths in `WatchList()`

## [2.1.0] - 2026-06-01

### Added

- **Polling mode** ‚Äî `WithPolling(true)` supplements fsnotify for NFS/FUSE/Docker
- **Debug logging** ‚Äî `WithDebug(logger)` for verbose structured logging
- **Circuit breaker middleware** ‚Äî closed/open/half-open fault tolerance
- **Throttle middleware** ‚Äî token-bucket rate limiting via `golang.org/x/time/rate`
- **File audit log middleware** ‚Äî write events to file for audit trail
- **Error middleware suite** ‚Äî rate limit, recovery, correlation, sanitization, batching
- **Symlink following** ‚Äî `WithFollowSymlinks(true)` during directory walking
- **Fuzz tests** ‚Äî expanded test coverage
- **goreleaser config** ‚Äî release automation
- **API stability policy** and troubleshooting guide
- **Code of conduct**

### Changed

- Go version bumped to 1.26.3

## [2.0.0] - 2026-05-23

### Breaking

- **MIT license** ‚Äî changed from proprietary
- **v2 module path** ‚Äî `github.com/larsartmann/go-filewatcher/v2`

### Added

- **Phantom types** ‚Äî `EventPath`, `RootPath`, `DebounceKey`, etc. for compile-time safety
- **slog.LogValuer** ‚Äî `Event` implements `slog.LogValuer`
- **JSON marshaling** ‚Äî full JSON support for `Event` and `Op`
- **Deduplicate middleware** ‚Äî drop duplicate events within a window
- **Batch middleware** ‚Äî batch events over a window or size threshold
- **Sliding-window rate limit** ‚Äî smooth rate limiting
