## G) Top #1 Question I Cannot Figure Out Myself

**Should v0.4.0 be a minor version bump (v2.2.0) or a major version bump (v3.0.0)?**

Arguments:

- **Minor (v2.2.0):** All changes are additive — new options (`WithGitignore`, `WithExcludePaths`, `WithMaxWatches`), new method (`Reset()`), new filter (`FilterGitignore`), new Stats fields. No breaking API changes. Existing code compiles and runs identically.
- **Major (v3.0.0):** The behavioral change is significant — `.gitignore` filtering is **enabled by default** (`gitignoreEnabled: true` in `New()`). Users upgrading may see different directories watched (fewer, because gitignored dirs are now skipped). This is a **semantic breaking change** even though the API is compatible.

The default-on behavior of gitignore filtering is the key question. Should we ship it as v2.2.0 with a clear changelog entry, or play it safe with v3.0.0?
