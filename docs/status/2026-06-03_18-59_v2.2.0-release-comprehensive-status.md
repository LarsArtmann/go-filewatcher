### Should we automate releases or keep them manual?

**The question:** Do you want me to set up a fully automated release pipeline (tag push → goreleaser → GitHub release → Go module available), or do you prefer the current manual process where we prepare, commit, tag, and push explicitly?

**Why I can't answer this:**
- `.goreleaser.yml` exists but has never been tested end-to-end
- There's no CI release workflow
- The previous 8 releases were all manual
- Automating requires decisions about: release notes source, pre-release vs stable, artifact signing, homebrew tap, etc.

**My recommendation:** Set up a `release.yml` GitHub Actions workflow triggered by `v*` tags that runs goreleaser. This is the standard Go ecosystem approach and eliminates the most error-prone step (forgetting to push tags, forgetting to create GitHub releases).
